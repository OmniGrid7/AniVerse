# AniVerse Articles

AniVerse Articles is a React-based anime article website where readers can explore anime writing, create a profile, publish their own posts, and recommend anime or manga with poster uploads. The app is designed as a small community writing space with a workspace-style sidebar, profile tools, and display settings.

## Features

- Browse anime articles by category.
- Search articles by title, author, topic, or content.
- Read full articles after creating a profile.
- Create and edit a reader/writer profile.
- Upload a profile photo.
- Publish new anime articles with your saved profile identity.
- Add anime or manga recommendations.
- Upload poster images for recommendations.
- Workspace sidebar with quick actions and user stats.
- Light, dark, and high-contrast themes.
- Font size, link style, and reduced-motion display controls.
- Local browser storage for profiles, posts, recommendations, and settings.

## Tech Stack

- React 18
- React DOM
- esbuild
- Node.js static server
- CSS for layout, responsive design, themes, and animations
- `localStorage` for browser-side persistence

## Getting Started

Install dependencies:

```bash
npm install
```

Build the site:

```bash
npm run build
```

Start the local server:

```bash
npm run dev
```

Open the website at:

```text
http://127.0.0.1:3000/
```

## Available Scripts

```bash
npm run dev
```

```bash
npm run start
```

## Main Pages

- `Home`: hero section, stats, mission section, and featured articles.
- `Explore`: full article feed with search and category filters.
- `Recommend`: add anime or manga recommendations with optional poster images.
- `Profile`: create or update profile details and profile photo.
- `Write`: publish a new anime article.

## Data Storage

This project stores user-created data in the browser using `localStorage`.

Stored data includes:

- Profile details
- Uploaded profile photo preview
- User-created articles
- Anime/manga recommendations
- Uploaded recommendation poster previews
- Theme and display settings

Because the data is browser-local, it will stay on the same browser/device unless local site data is cleared.

## Notes

The `src` folder contains the editable source code. The `dist` folder is generated by the build process and is what the local server serves.

If you change files inside `src`, run:

```bash
npm run build
```

Then refresh the browser to see the updated site.
